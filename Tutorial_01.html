1. Data Types & Data Structures¶

1.1. Basic data types¶

Human uses a variety of symbols and languages to store data. In contrast, a computer can only read binary numbers. It is necessary to encode our data into binary before the computer can work with. For different kinds of data, we need different encoding schemes to convert them into the same set of binaries (e.g. 64-bit system = every operation works on a binary number with 64 digits). Then by telling the computer what the binary number is AND what the encoding scheme it used, the computer can determine by itself what are the valids operations for that data.

These different encoding schemes are exactly corresponding to the different data types we use in programming. Here lists out some basic data types you must be familiar with.

1.1.1 Int¶

Intergers. The basic encoding is just by decimal-to-binary conversion. However a lot of times we want to work with negative intergers, the scheme can be further splited into signed int and unsigned int, which in the former scheme, one of the digits is reserved to represent whether the number has positive or negative sign.

1.1.3 Bool¶

Boolean, i.e. True or False. Although a boolean can be represented simply by 1 digit, for efficienct reading/writing from the computer memory, most of the time they are stored using 1 byte (=8 bits) which is the smallest unit in memories of most computers.

1.1.4 Char¶

Characters, including every alphabets, numerals and symbols we can type on our keyboard. There a too many different encoding schemes, depending on the languages and usages. Here are two very common schemes you should have heard of:

  • ASCII - Encoded every english alphabet, 0-9, and symbols you can find on english keyboard into 8-bit binaries. You may see here for the table of conversion.

  • Unicode - A standardized library that tries to include all characters from every language, all symbols and even emojis. It is pretty much the standard encoding in modern communication. To learn more, you may visit their main website here.

1.2. Data types of linear collections¶

In order to work with a large quantity of data, we need more advanced data structure to store the data. They may be classified as follow:

  • Linear collection - Each object is at most connected to two other objects - the "previous" and the "next". E.g. array, hash table, linked list, stack, queue, etc.

  • Non-linear collection - Each object can be connected to a collection of objects and does not have to come one after another. E.g. tree, graph, etc.

Understanding the pro and cons of each of the data structure is fundamental to design effcient algorithms. We will focus on linear collection this week.

1.2.1 Array¶

A sequence of data stored in a *continuous section* of the computer's physical memory. It is the simplest structure for storing a collection of data. Here is a simplified illustration of how an array is stored (the exact implementation varies in different languages):

Because the data must be arranged in the order of index / physical memory address, array has the following advantage and disadvantage:

  • Accessing elements is instant if the index is known, by just calculating (address of first element) + index once (i.e. $O(1)$).
  • Insertion/deletion of elements in the middle of an array are slow, because you need to shift the memory address of all subsequent elements. (# of shift ~ # of element, so $O(n)$)
  • Increasing the length of an array is memory inefficient, because there may not be enough empty memory address after the array for placing new elements. The standard solution is first find a new location that can hold the increased-size array, then copy the all the elements to the new memory address.

1.2.2 String¶

Basically the same as an array, but it can only contain char types data. Also depending on the languages, there will be more operations developed specifically for string type data but cannot be used on normal array.

1.2.3 Hash Table (dictionary)¶

An unordered structure that store data in the form of key-value pairs. The main component behind is by having a hash function $y=H(x)$ which, if ideally:

  • Computation is very fast.
  • Different input $x$ always gives different output $y$.
  • The values of $y$ are integers and are within a certain range.
  • Must be deterministic, i.e. no random variable in the function.

What kinds of functions are used as hash function is itself a topic in number theory and is beyond our scope. Here only describes the basic idea of how hash table is can be stored in the comuter memory:

  • All the keys are stored in a list-liked data structure. Whether they are sorted depends on the language.
  • Each key must be unique.
  • Operate the hash function on the key yields the memory address where the value is stored.

Hash table has the following advantages and disadvantages:

  • Checking if a key exists, or inquiring the value that paired to the key, are instant, by just calculating $H(\text{key})$ once (i.e. $O(1)$).
  • Insertion/deletion depends on what data structure is used to store the keys. $O(1)$ if the keys are not required to sort, normally $O(n\log n)$ if requires sorting.

1.3. Linked lists¶

A linked list is an advanced linear data structure that stores each data as an individual object (i.e. node).

  • Each data/node does not need to be stored in an organized order in the computer's memory.
  • The connection of data/nodes are by "pointer". i.e. Every data/object contains an extra piece of info, telling the address of its neighbour data/objects in the computer's physical memory.

Using singly-linked list as an example:

Linked list has the following advantages and disadvantages:

  • Accessing elements is slow, since the data/nodes do not carry the info of their order in the list. One has to iterate $k$ times to reach the $k^{th}$ element. Therefore it is $O(n)$.
  • Insertion/deletion are fast if we already know where the element is, since one only needs to change the pointing address of the neighbour of the inserted/deleted data. So theoretically they can be as fast as $O(1)$.

There are a lot of variations of linked lists, include:

  • Singly linked list
  • Doubly linked list
  • Circular list
  • Skip list
  • ...

and many more. In particular we are more interested in these 2 because of their usefulness:

  • Stack - Essentially a linked list, but insertion/deletion follow the Last In First Out (LIFO) rule. i.e. restricted to the last element only (so called "push" and "pop").

  • Queue - Essentially a linked list, but insertion/deletion follow the First In First Out (FIFO) rule. i.e. insert is restricted to the last and delete is restricted to the first element. They are also called "push" and "pop", but you may see them being called "enqueue" and "dequeue".

2. Implementation in Python¶

Unfortunately in Python, a higher level language, basically everything is coded internally as objects, which means:

  • Cannot use the binary representation of the basic data types to speed up calculations.
  • A lot of data structures are coded in the defintions of the data type. You don't have to work them out (or even understand how the algorithms run in the background)

(If you are looking for full control, please learn C++, which is what Python is compiled from.)

However this course is working around with Python so you need to at least know some common data structures in Python and how they run like those fundamental data structures.

2.1. Array-like structures¶

These are the array-like structures available in vanilla Python:

  • List - It is stored in computer's memory like ordinary array, but added methods like .append() and .pop() so that you can use it like a stack or queue. Yet its operations' efficiency are the same as array.
In [1]:
my_list = [True, "I love Python", 5051]
  • collections.deque - A special doubly-linked list, provided by the Python standard library collections, that is particularly optimized for inserting/deleting elements at both ends of the list, but not elements in the middle of the list.
In [2]:
from collections import deque
my_queue = deque([True, "I love Python", 5051])

Also worth mentioning - Numpy array, although not a vanilla data type in Python, will probably be the most frequent type you use in other courses.

  • Numpy array - They are stored in computer's memory like ordinary array, however in an optimized way that requires less memory and for carrying out faster elementwise operations. You should always declare a list as a Numpy array if you are doing numerical computation with Numpy.
In [3]:
import numpy as np
my_npArray = np.array([5003, 5051, 5056])

2.3. Other not-so-common data structures¶

Other than list, Python provides others kind of data structures that can come handy in different scenarios.

Structures for sequential data:¶

Prefered if the order of data is important. List also belongs to this category.

  • Tuple - Same as a list except that its content cannot be changed after it is declared.
  • Range - Just like a tuple but its content must be a sequence of number.
  • String - Just like a tuple but its content would be interpreted as texts (~ char type in C).
In [5]:
my_tuple = (True, "I love Python", 5051)
my_range = range(1,5051, -1)
my_string = "I love Python"

Structures for non-sequential data:¶

Structures that are highly optimized for checking if a specific item is in the data.

  • Dictionary - Similar to a hash table.
  • Set - Similar to a dictionary, except that the entries in the values' table can only be true or false (i.e. key is created or not).
In [6]:
my_dict = {"attendance": True, "opinion": "I love Python", "course": 5051}
my_set = {True, "I love Python", 5051}

To learn more, you may read the documentation of Python on data structures and built-in data types.
